# Contributing To SignalPlane

Thank you for contributing to SignalPlane.

## Development Values

- Prefer open standards.
- Keep user workflows clear and testable.
- Make telemetry relationships visible.
- Treat privacy and security as product requirements.
- Avoid unnecessary lock-in.
- Document operational tradeoffs.

## Contribution Types

- Product requirements.
- Documentation.
- Collectors and integrations.
- Backend services.
- Frontend UI.
- Storage and query improvements.
- Alerting and workflow features.
- Testing and release automation.

## Pull Request Expectations

- Describe the user-facing change.
- Include tests for behavioral changes.
- Update documentation when workflows change.
- Avoid unrelated refactors.
- Keep sensitive data out of fixtures.
- Add screenshots for UI changes.
