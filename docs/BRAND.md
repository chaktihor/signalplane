# SignalPlane Brand

## Name

**SignalPlane**

## Positioning

SignalPlane is the open telemetry command plane for software teams. It brings metrics, logs, traces, topology, incidents, user experience, runtime risk, and automation into one coherent product.

## Tagline Options

- The open command plane for observability.
- Turn telemetry into action.
- One plane for signals, systems, and reliability.
- Open observability with operational intelligence.

## Brand Attributes

- Clear
- Technical
- Trustworthy
- Fast
- Open
- Practical
- Extensible

## Voice

SignalPlane should sound calm, precise, and operationally useful. Product copy should help users understand what is happening, why it matters, and what to do next.

Avoid hype-heavy language. Prefer concrete wording such as:

- "Investigate incident"
- "View affected services"
- "Compare deployment impact"
- "Create SLO"
- "Route alert"
- "Mask sensitive fields"

## Naming Conventions

- Platform name: **SignalPlane**
- CLI name: `spctl`
- Default collector bundle: **SignalPlane Collector**
- Agent name: **SignalPlane Node Agent**
- Query language placeholder: **Signal Query**
- Workflow engine placeholder: **SignalFlow**
- Topology engine placeholder: **Signal Graph**

## Visual Direction

- Use a restrained technical interface with high information density.
- Prefer neutral backgrounds with strong status colors.
- Use color meaningfully for severity, health, ownership, and data type.
- Avoid decorative gradients, oversized marketing cards, and purely ornamental visuals inside the product.
